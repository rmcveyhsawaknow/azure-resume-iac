# [Phase 0] Verify Azure SP credential

## Description

Verify the Azure Service Principal (SP) credential stored in GitHub Secrets is valid and has not expired. Confirm the SP has the necessary RBAC roles (Contributor or equivalent) on all target subscriptions and resource groups. This is a prerequisite for all IaC and Azure deployment tasks.

## Acceptance Criteria

- [ ] SP credential retrieved from GitHub Secrets
- [ ] Successful `az login --service-principal` using the credential
- [ ] Confirmed RBAC role assignments on target subscription
- [ ] Documented SP app ID, tenant ID, and expiration date
- [ ] If expired, new credential generated and GitHub Secret updated

## Dependencies

None — this is a root task.

## Copilot Instructions Reference

N/A — requires Azure Portal/CLI access and credential management.

## Notes

If the SP credential is expired or invalid, all downstream Azure tasks will be blocked. Prioritize immediate resolution.
