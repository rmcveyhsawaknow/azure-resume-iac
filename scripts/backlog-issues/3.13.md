# [Phase 3] Migrate backend App Insights to workspace-based

## Description

The backend Application Insights resource (`cus1-resumectr-prod-v1-ai`) is in **Classic mode** with `IngestionMode: Disabled`. Classic mode is deprecated and Microsoft has announced its retirement. This was identified as finding F13 during Phase 0 gap analysis (Issue #17).

Migrate to workspace-based mode by:
1. Creating or referencing a Log Analytics workspace in Bicep
2. Updating the App Insights resource to workspace-based mode
3. Setting `IngestionMode` to `LogAnalytics`
4. Updating the Function App connection string to full format (with `IngestionEndpoint`)

## Acceptance Criteria

- [ ] Backend App Insights resource converted to workspace-based mode in Bicep
- [ ] Log Analytics workspace created or referenced
- [ ] IngestionMode set to `LogAnalytics`
- [ ] Function App `APPLICATIONINSIGHTS_CONNECTION_STRING` updated to full format
- [ ] Telemetry verified flowing after Function App restart

## Dependencies

- Task 3.1: Update dev workflow variables

## Copilot Instructions Reference

Follow Azure Verified Modules patterns for `Microsoft.Insights/components` with `WorkspaceResourceId` property.
