# [Phase 5] Address frontend instrumentation key hardcoding

## Description

The frontend JavaScript file `azure_app_insights.js` contains a **hardcoded Application Insights instrumentation key**. This is a known security debt item per `docs/KNOWN_ISSUES.md`. The key was identified as finding F20 during Phase 0 gap analysis (Issue #20).

Options to address:
1. **Token replacement at deploy time**: CI/CD workflow replaces a placeholder with the actual key during `buildDeployFrontend` job
2. **Environment-specific config**: Use a separate config file loaded at runtime, replaced during deployment
3. **Accept and document**: If the iKey exposure risk is acceptable, document the decision and move on

The iKey alone does not grant access to read data — it only allows writing telemetry. The risk is telemetry injection, not data exfiltration.

## Acceptance Criteria

- [ ] Instrumentation key no longer committed in plain text to source control
- [ ] Or: risk acknowledged and documented in `docs/KNOWN_ISSUES.md` with rationale
- [ ] Frontend telemetry verified still working after any changes
- [ ] CI/CD workflow updated if token replacement approach is used
