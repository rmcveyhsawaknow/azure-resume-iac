# [Phase 5] Set Key Vault secret expiration dates

## Description

Key Vault secrets have **no expiration dates** configured, meaning they never rotate automatically and there is no visibility into secret age. This was identified as finding F15 during Phase 0 gap analysis (Issue #15).

Add expiration dates to all Key Vault secrets in the Bicep template. Recommended: 1-year expiration with renewal process documented.

## Acceptance Criteria

- [ ] All Key Vault secrets have expiration dates set in Bicep template
- [ ] Expiration policy documented (recommended: 1 year)
- [ ] Alert or monitoring for upcoming expirations (optional, document as future improvement if not implemented)

## Dependencies

None — can be done during final cleanup.

## Copilot Instructions Reference

Follow Azure Verified Modules patterns for `Microsoft.KeyVault/vaults/secrets` `properties.attributes.exp` (Unix epoch timestamp).

## Notes

Source: Phase 0 gap analysis finding F15 (Issue #15). Secrets include Cosmos DB connection strings and Function App keys.
