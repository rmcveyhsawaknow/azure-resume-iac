# [Phase 3] Update backend App Insights connection string format

## Description

The backend Function App has an `APPLICATIONINSIGHTS_CONNECTION_STRING` that uses the older format with only `InstrumentationKey=...`, missing the `IngestionEndpoint` and `LiveEndpoint` fields. This was identified as finding F27 during Phase 0 gap analysis (Issue #17).

Update the app setting to use the full connection string format provided by the workspace-based App Insights resource.

## Acceptance Criteria

- [ ] Function App `APPLICATIONINSIGHTS_CONNECTION_STRING` includes `IngestionEndpoint` and `LiveEndpoint`
- [ ] Connection string sourced from Bicep output (not hardcoded)
- [ ] Telemetry verified flowing after update

## Dependencies

- Task 3.13: Migrate backend App Insights to workspace-based (provides the new connection string)

## Copilot Instructions Reference

Reference the Bicep output for `Microsoft.Insights/components` `connectionString` property.
