# [Phase 3] Add production environment protection rules

## Description

The `production` GitHub environment has **no protection rules** — no required reviewers, no wait timer, and no deployment branch restrictions. Any branch can currently trigger a deployment to production. This was identified as finding F6 (Critical) during Phase 0 gap analysis (Issue #23).

Configure the `production` environment with:
- At least 1 required reviewer
- Deployment branch policy restricting to `main` branch only
- Optional: 5-minute wait timer for cancellation window

## Acceptance Criteria

- [ ] Production environment has at least 1 required reviewer configured
- [ ] Deployment restricted to `main` branch only via branch policy
- [ ] Protection rules verified via `gh api repos/{owner}/{repo}/environments/production`
- [ ] Wait timer configured (optional, recommended 5 minutes)

## Dependencies

None — can be done anytime before production deployment in Phase 4.

## Copilot Instructions Reference

N/A — requires GitHub repository admin access for environment configuration.
