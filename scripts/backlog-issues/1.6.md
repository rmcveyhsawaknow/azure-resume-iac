# [Phase 1] Verify Cosmos DB data

## Description

Verify the Cosmos DB visitor counter data is intact and in the expected format. Confirm the document structure matches what the upgraded Function App expects. If the data format needs updating, prepare a migration script.

## Acceptance Criteria

- [ ] Visitor counter document exists in the expected container
- [ ] Document structure matches the Function App's model class
- [ ] Partition key value is correct
- [ ] Data is readable via Cosmos DB Data Explorer or SDK
- [ ] If format differs, migration script prepared
