# [Phase 1] Test function locally

## Description

Test the upgraded Function App locally using Azure Functions Core Tools. Verify the function starts, accepts HTTP requests, connects to Cosmos DB (or local emulator), and returns the expected response. This is the final local validation before deploying to Azure.

## Acceptance Criteria

- [ ] Azure Functions Core Tools v4 installed locally
- [ ] `func start` launches the Function App without errors
- [ ] HTTP trigger endpoint responds to requests
- [ ] Cosmos DB connection works (emulator or remote)
- [ ] Visitor counter increments correctly
- [ ] Response format matches frontend expectations

## Dependencies

- Task 1.2: Upgrade .NET runtime (.NET Core 3.1 → .NET 8)
- Task 1.3: Upgrade Functions version (v3 → v4)
- Task 1.4: Update NuGet packages
- Task 1.5: Update test project

N/A — requires local execution environment and manual testing.

## Notes

Use `local.settings.json` with a connection to the actual Cosmos DB (dev) or the local emulator for testing.
