# [Phase 3] Enable Key Vault soft delete

## Description

The Key Vault (`cus1-resume-prod-v1-kv`) has **soft delete disabled**, which means deleted secrets cannot be recovered. This was identified as finding F14 during Phase 0 gap analysis (Issue #15). Purge protection is already enabled.

Update the Bicep template to set `enableSoftDelete: true` with an appropriate retention period (default 90 days).

## Acceptance Criteria

- [ ] Bicep template updated: `enableSoftDelete: true` on Key Vault
- [ ] Soft delete retention period set (default 90 days)
- [ ] Verified via `az keyvault show` after deployment
- [ ] Purge protection remains enabled

## Dependencies

- Task 3.1: Update dev workflow variables

## Copilot Instructions Reference

Follow Azure Verified Modules patterns for `Microsoft.KeyVault/vaults` `properties.enableSoftDelete`.

## Notes

Source: Phase 0 gap analysis finding F14 (Issue #15). Note: once soft delete is enabled on a Key Vault, it cannot be disabled. This is a one-way change.
