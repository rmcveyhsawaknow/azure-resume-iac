# Phase Retrospectives

This directory holds committed retrospective summaries generated at each phase boundary using the AgentGitOps workflow.

## Files

| File | Phase | Status |
|---|---|---|
| `phase-0-retrospective.md` | Phase 0 — Assessment | ✅ Complete |
| `phase-1-retrospective.md` | Phase 1 — Fix Function App | ✅ Complete |
| `phase-2-retrospective.md` | Phase 2 — Content Update | ✅ Complete (milestone closure pending) |
| `phase-3-retrospective.md` | Phase 3 — Dev Deployment | ✅ Complete |
| `phase-4-retrospective.md` | Phase 4 — Prod Deployment | Pending |
| `phase-5-retrospective.md` | Phase 5 — Cleanup & Docs | Pending |

## How They're Generated

Each retrospective is generated by [`scripts/generate-phase-retrospective.sh`](../../scripts/generate-phase-retrospective.sh):

```bash
# Dry run (preview only)
./scripts/generate-phase-retrospective.sh 0 --dry-run

# Generate and post to retrospective issue
./scripts/generate-phase-retrospective.sh 0
```

The script collects milestone issues, PR/commit stats, Human vs Copilot KPIs, and gap analysis findings, then writes the report here and optionally posts it as a comment on the matching retrospective GitHub issue.

## Workflow

1. Assign phase issues to their milestone
2. Run the generator script at phase end
3. Commit the generated `.md` file
4. Post the report to the retrospective issue
5. Close the milestone
