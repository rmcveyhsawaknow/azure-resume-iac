# [Phase 1] Set FtpsState to Disabled on Function App

## Description

The Function App currently has `FtpsState: AllAllowed`, which permits unencrypted FTP access. This is a security risk identified during Phase 0 gap analysis (finding F12, Issue #13). Update the Bicep template and Function App configuration to set `ftpsState: 'Disabled'`.

## Acceptance Criteria

- [ ] Function App `ftpsState` set to `Disabled` in Bicep template
- [ ] Verified via `az functionapp config show` after deployment
- [ ] No FTP/FTPS access possible to the Function App

## Dependencies

- Task 1.3: Upgrade Functions version (v3 → v4)

## Copilot Instructions Reference

Follow Azure Verified Modules patterns for Function App `siteConfig` properties in Bicep templates.

## Notes

Source: Phase 0 gap analysis finding F12 (Issue #13). FtpsState should be `Disabled` since deployments use GitHub Actions zip deploy, not FTP.
