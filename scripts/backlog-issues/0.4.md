# [Phase 0] Assess Cosmos DB

## Description

Assess the Cosmos DB account(s) used by the resume site. Document the account name, API type, databases, containers, partition keys, and throughput settings. Verify that the visitor counter data exists and is accessible.

## Acceptance Criteria

- [ ] Cosmos DB account name, resource group, and API type documented
- [ ] Databases and containers listed with partition key and throughput
- [ ] Visitor counter data verified (record exists and is readable)
- [ ] Connection string availability confirmed in Key Vault or app settings
- [ ] Cost/throughput assessment completed

## Dependencies

- Task 0.1: Verify Azure SP credential

## Copilot Instructions Reference

Copilot can assist with scripting `az cosmosdb` commands for inventory.

## Notes

The Cosmos DB stores the visitor counter. Ensure the data format is compatible with the Function App after the .NET upgrade in Phase 1.
