# [Phase 1] Upgrade .NET runtime (.NET Core 3.1 → .NET 8)

## Description

Upgrade the Function App backend from .NET Core 3.1 to .NET 8 (LTS). This involves updating the target framework in the `.csproj` file, updating API calls that have changed between versions, and resolving any breaking changes. This is a major version upgrade spanning multiple .NET generations.

## Acceptance Criteria

- [ ] `.csproj` updated to target `net8.0`
- [ ] All breaking API changes resolved
- [ ] Code compiles without errors on .NET 8
- [ ] Isolated worker model adopted (if applicable)
- [ ] Local build succeeds with `dotnet build`

## Dependencies

- Task 1.1: Diagnose root cause

## Copilot Instructions Reference

Copilot can fully assist with `.csproj` edits, API migration, and code updates. Reference `.github/copilot-instructions.md` for backend conventions.

## Notes

.NET Core 3.1 → .NET 8 is a multi-generation jump. Key changes include new hosting model, updated dependency injection, and changes to `Startup.cs` patterns.
