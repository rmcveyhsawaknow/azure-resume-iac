# [Phase 3] Update storage account min TLS to 1.2

## Description

All 4 storage accounts currently have `minimumTlsVersion: TLS1_0`, which permits insecure TLS 1.0 and 1.1 connections. This was identified as finding F11 during Phase 0 gap analysis (Issue #16).

Update the Bicep template to set `minimumTlsVersion: 'TLS1_2'` on all storage account resources (frontend static sites and Function App backing storage).

## Acceptance Criteria

- [ ] Bicep template updated: `minimumTlsVersion: 'TLS1_2'` for all storage accounts
- [ ] Verified via `az storage account show` after deployment
- [ ] No impact on static website serving (TLS 1.2 is universally supported by modern browsers)

## Dependencies

- Task 3.1: Update dev workflow variables

## Copilot Instructions Reference

Follow Azure Verified Modules patterns for storage account `properties.minimumTlsVersion` in Bicep. Ensure encryption at rest is also configured per copilot-instructions.md.

Source: Phase 0 gap analysis finding F11 (Issue #16). Affected accounts: `cus1resumeprodv1sa`, `cus1resume2prodv1sa`, `cus1resume3prodv1sa`, `cus1resumectrprodv1sa`.
